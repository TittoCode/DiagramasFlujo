Empecemos

Introducción a los Diagramas de Flujo

Domina la representación visual de algoritmos, la habilidad técnica que separa a los que "pican código" de los verdaderos programadores.

¿Qué son?

Es el mapa visual de tu lógica. Utiliza símbolos universales para mostrar el camino que siguen los datos, permitiéndote resolver problemas complejos paso a paso antes de tocar el teclado.

En la Facultad

Es el lenguaje base en Programación. Antes de usar Python o otro lenguaje de programación, los diagramas te enseñan a pensar algorítmicamente, asegurando que tus soluciones tengan una estructura sólida y aprobable.

¿Por qué aprender?

Te ahorra horas de frustración frente a la pantalla. Al diagramar, detectas errores de lógica tambien te ayuda a entender mejor la logica de programacion.

Símbolos Fundamentales

El "alfabeto" visual que todo estudiante de sistemas debe dominar para aprobar Programación I.

Terminal

Indica dónde comienza y termina tu algoritmo. Siempre debe haber un solo Inicio y al menos un Fin claramente marcados.

Proceso

Se usa para realizar operaciones matemáticas o asignar valores a variables. Ejemplo: suma = a + b.

Decisión

Evalúa una condición lógica. Tiene obligatoriamente dos salidas: una por Verdadero (Sí) y otra por Falso (No).

Entrada (Leer)

Representa el ingreso de datos por parte del usuario mediante el teclado. Se asocia con la instrucción Leer.

Salida (Mostrar)

Muestra los resultados o mensajes en pantalla. Es el símbolo que usas para imprimir el famoso "Hola Mundo".

Conector

Sirve para enlazar partes del diagrama en la misma página cuando te quedas sin espacio vertical. ¡Mantiene el orden!

Bucle FOR (Para)

Repetición controlada. Úsalo cuando sabes exactamente cuántas veces se debe repetir el proceso (Ej: del 1 al 10).

Bucle WHILE (Mientras)

Evalúa la condición antes de entrar. Si no se cumple al inicio, el código de adentro nunca se ejecuta.

DO-WHILE (Hacer hasta)

Evalúa la condición al final. Garantiza que el proceso se ejecute al menos una vez antes de preguntar.

Estructuras de Control

Los cimientos de la lógica: aprende cómo decidir caminos y repetir tareas de forma inteligente.

Condicional IF-ELSE

Toma de Decisiones

Es la bifurcación del camino. Permite que el algoritmo elija entre dos opciones (V/F). Ejemplo práctico: Verificar si un alumno tiene nota > 51 para marcarlo como "Aprobado".

Bucle FOR (Para)

Ciclo Controlado

Ideal cuando ya conoces el número exacto de vueltas. Usa una variable (contador) que se incrementa automáticamente. Ejemplo: Imprimir las tablas de multiplicar del 1 al 10.

Bucle WHILE

Pre-condición

El "guardián" de la entrada. Si la condición no se cumple de entrada, el código de adentro nunca se ejecuta. Ejemplo: Pedir una contraseña hasta que sea correcta.

Bucle DO-WHILE

Post-condición

Ejecuta la acción y después pregunta. Garantiza al menos una ejecución. Ejemplo ideal: Menús de cajero automático donde primero ves las opciones y luego decides si salir.

Fundamentos de Programación

Domina los pilares lógicos que permiten construir cualquier software, desde una aplicación móvil hasta una inteligencia artificial.

Imagina una variable como una "caja" etiquetada donde guardas información que puede cambiar, mientras que una constante protege valores que deben permanecer fijos.

let usuario = "Alex"; // Variable
const PI = 3.1416; // Constante

Tipos Primitivos

String (Texto) Number (Números) Boolean (V/F) Array (Listas)

Operadores de Verdad

AND (&&) Ambos deben ser ciertos
OR (||) Uno de los dos basta
NOT (!) Invierte el resultado

El operador Módulo (%)

No devuelve el cociente, sino el residuo de una división. Es clave para detectar números pares o ciclos.

10 % 3 = 1

Determinan el flujo de ejecución: si el programa debe saltar una instrucción o repetir una tarea.

Condicionales (If/Else)

Ejecutan código solo si se cumple una condición. "Si llueve, lleva paraguas".

Bucles (For/While)

Repiten una acción mientras la condición sea válida. Ideal para procesar listas.
